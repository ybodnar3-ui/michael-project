# AI Automation Diagnostic — Design Document

**Дата:** 2026-06-22
**Автор:** Yura (ybodnar3-ui) + Claude
**Статус:** Затверджено до реалізації (фазова розбивка нижче)
**Репозиторій:** `ybodnar3-ui/michael-project`

---

## 1. Суть продукту (одним абзацом)

Веб-застосунок-діагностика. Власник бізнесу заходить за посиланням, спілкується з AI, який ставить йому адаптивні питання про його сферу, і в кінці отримує **персональний звіт «що саме у твоєму бізнесі можна автоматизувати за допомогою AI»**. Кожна можливість сформульована так, що бізнес одразу розуміє: «ось із цим запитом я йду до спеціаліста». Контакт ліда + повний звіт автоматично прилітають власнику продукту (Yura).

Це **не разовий продукт під одного клієнта**, а **власний актив-воронка лідогенерації**: один лінк, який можна слати будь-якому бізнесу. Сфера бізнесу — параметр, а не зашите в код, тому той самий продукт працює для будь-якої ніші.

---

## 2. Бізнес-модель і позиціонування

- **Власник активу — Yura.** Бренд, домен, дані, ліди — все його. Бізнесмен-замовник просто перший користувач.
- **Цінність для бізнесу-користувача:** безкоштовний персональний AI-аудит автоматизації за 5 хвилин. Вау-ефект = красивий конкретний звіт.
- **Цінність для Yura:** холодний бізнес → теплий лід із готовим брифом. Замість «давай я тобі щось продам» — людина приходить сама з «хочу оце, оце і оце, скільки коштує».
- **Канал поширення:** одне посилання. Кидається в месенджери, розсилки, рекламу, кому завгодно.

---

## 3. Користувацький сценарій (end-to-end)

1. **Лендинг.** Бізнесмен відкриває посилання. Бачить брендований екран: заголовок («Дізнайся за 5 хвилин, що у твоєму бізнесі можна автоматизувати»), коротке пояснення, кнопка «Почати».
2. **Старт.** AI вітається, питає сферу/нішу бізнесу (вільний текст).
3. **Адаптивне інтерв'ю.** AI ставить ~8–15 питань **по одному за раз**, підлаштованих під сферу. Покриває: що робить бізнес, розмір команди, ключові процеси, де витікають час/гроші, що роблять руками, як іде потік клієнтів, якими інструментами користуються тощо.
4. **Завершення збору.** AI розуміє, що інформації достатньо, і повідомляє: «Готую твій персональний звіт».
5. **Збір контакту (lead gate).** Перед показом звіту: «Куди надіслати звіт і як до тебе звертатись?» → ім'я + контакт (email / телефон / Telegram).
6. **Звіт.** Структурований аудит на гарній сторінці: список можливостей автоматизації, кожна з описом проблеми, рішення, орієнтовного ефекту і **готовим формулюванням запиту до спеціаліста**. Зверху — пріоритетна рекомендація. Знизу — CTA «Обговорити впровадження».
7. **Маршрутизація ліда.** Контакт + повний звіт автоматично летять Yura (Telegram-бот + email).

## 4. Архітектура

```
┌─────────────────────────────────────────────────────────────┐
│                     Браузер користувача                      │
│   Лендинг → Чат-інтерфейс → Lead gate → Сторінка звіту       │
└───────────────────────────────┬─────────────────────────────┘
                                 │ HTTPS (тільки наш бекенд)
                                 ▼
┌─────────────────────────────────────────────────────────────┐
│                Next.js (App Router) — сервер                 │
│  Route handlers:                                             │
│   • POST /api/chat      — крок діалогу (виклик Claude)       │
│   • POST /api/report    — генерація структурованого звіту    │
│   • POST /api/lead      — збереження контакту + сповіщення   │
│                                                              │
│  Захист: rate-limit, message caps, Turnstile-перевірка       │
└───────────┬───────────────────────┬─────────────────────────┘
            │                       │
            ▼                       ▼
   ┌─────────────────┐     ┌──────────────────────┐
   │  Claude API     │     │  БД (Postgres)       │
   │  (server-side,  │     │  sessions, leads     │
   │   ключ прихов.) │     └──────────────────────┘
   └─────────────────┘                │
                                      ▼
                          ┌──────────────────────┐
                          │ Сповіщення Yura       │
                          │ Telegram-бот + email  │
                          └──────────────────────┘
```

**Ключовий принцип ізоляції:** API-ключ Claude живе **тільки на сервері**. Браузер ніколи його не бачить. Уся комунікація з Claude — через наші route handlers.

## 5. Компоненти системи (з межами відповідальності)

### 5.1 «Мозок» — діагностична методологія (системний промпт)
- **Що робить:** інструктує Claude, як вести інтерв'ю бізнесу: які зони прощупати, як зв'язати болі → можливості автоматизації, коли зупинитись, як сформулювати запит до спеціаліста.
- **Це головне IP продукту.** Окремий, ретельно спроєктований артефакт.
- **Залежить від:** нічого (чистий текст-інструкція).

### 5.2 Conversational endpoint (`POST /api/chat`)
- **Що робить:** приймає історію діалогу + нове повідомлення → викликає Claude → повертає наступне питання. Слідкує за лімітами (макс. повідомлень, довжина).
- **Залежить від:** системний промпт, Claude SDK, store сесії.

### 5.3 Report generator (`POST /api/report`)
- **Що робить:** на основі зібраного діалогу генерує **структурований JSON-звіт** (через structured output / JSON schema).
- **Залежить від:** історія сесії, Claude SDK, схема звіту.

### 5.4 Lead capture & routing (`POST /api/lead`)
- **Що робить:** зберігає контакт + сесію + звіт у БД, шле сповіщення Yura.
- **Залежить від:** БД, Telegram-бот, email-сервіс.

### 5.5 Frontend (лендинг + чат + звіт)
- **Що робить:** брендований UI всіх кроків сценарію. Має виглядати як справжній продукт.
- **Залежить від:** endpoints вище.

### 5.6 Захист від зловживань
- **Що робить:** Cloudflare Turnstile (невидима капча), rate-limit по IP, ліміти повідомлень/довжини, бюджетний cap на API-ключі.
- **Залежить від:** обгортає всі endpoints.

## 6. Модель даних

**Session**
```
id            uuid
created_at    timestamp
sphere        text            -- сфера бізнесу
messages      jsonb           -- історія діалогу [{role, content}]
status        enum            -- in_progress | report_ready | lead_captured
report        jsonb | null    -- структурований звіт
ip_hash       text            -- для rate-limit (хеш, не сирий IP)
```

**Lead**
```
id            uuid
session_id    uuid -> Session
name          text
contact       text
channel       enum            -- email | phone | telegram
created_at    timestamp
notified      boolean         -- чи відправлено Yura
```

## 7. Структура звіту (JSON schema, орієнтовно)

```
{
  business_summary: string,              // 1-2 речення: як AI зрозумів бізнес
  automation_opportunities: [
    {
      title: string,                     // напр. "Автоматична кваліфікація лідів"
      problem: string,                   // який біль вирішує
      solution: string,                  // що саме автоматизуємо
      ai_capability: string,             // яка AI-функція це робить
      estimated_impact: string,          // орієнтовний ефект (час/гроші)
      request_to_specialist: string      // готовий запит "хочу X"
    }
  ],
  priority_recommendation: string,       // з чого почати
  next_step: string                      // CTA текст
}
```

## 8. Технологічний стек (рішення)

| Шар | Вибір | Чому |
|---|---|---|
| Frontend + Backend | **Next.js 14+ (App Router), TypeScript, Tailwind** | Один деплой, route handlers ховають ключ, найкраще під веб-продукт |
| LLM | **Claude Sonnet 4.6** (`claude-sonnet-4-6`) для діалогу; опційно Opus на звіт | Баланс ціна/якість; діалог не потребує Opus |
| Claude інтеграція | **`@anthropic-ai/sdk`**, structured outputs, prompt caching | Кешування системного промпта = −90% на повторюваному контексті |
| БД | **Hosted Postgres (Neon або Supabase, free tier)** | Serverless-сумісно з Vercel, безкоштовно на старті |
| Сповіщення | **Telegram-бот** (+ email через Resend) | Yura в Telegram-екосистемі; миттєво |
| Хостинг | **Vercel** | Нативно під Next.js, простий деплой |
| Захист ботів | **Cloudflare Turnstile** | Безкоштовно, невидимо |

**Зафіксовані припущення (можна переглянути):**
- Звіт спершу — **гарна сторінка на екрані**. PDF-експорт — у беклог (YAGNI на MVP).
- Мова інтерфейсу/діалогу — **українська** на старті (модель легко перемкнути на іншу пізніше).
- Одна модель (Sonnet 4.6) на весь флоу для простоти MVP; ескалація на Opus для звіту — тюнінг пізніше.

## 9. Захист і контроль витрат (зведено)

1. **Обмежений діалог, а не вільний чат** — макс. ~25 повідомлень/сесія, потім звіт і стоп.
2. **Ліміт довжини повідомлення** (~2000 символів).
3. **Rate-limit по IP** — напр. 2-3 сесії/добу з адреси.
4. **Звіт за контакт** — lead gate відсіює «потика» і ловить ліда.
5. **Дешева модель + кешування промпта.**
6. **Бюджетний cap на API-ключі** у консолі Anthropic.
7. **Cloudflare Turnstile** проти ботів.

Реальна вартість 1 повної розмови: **~$0.15–0.35** (Sonnet) / **~$0.05–0.10** (Haiku). Окупається з першого замовлення.

## 10. Фазова розбивка реалізації

Кожна фаза дає **щось робоче й перевіряюване**. Йдемо по черзі; кожна фаза перед стартом отримує свій детальний план.

### Фаза 0 — Фундамент і скелет
- Структура репозиторію, Next.js scaffold (TS + Tailwind), env-конфіг, базовий деплой на Vercel.
- **Результат:** порожній застосунок запускається локально і деплоїться.

### Фаза 1 — Діагностичний мозок (промпт + chat API)
- Проєктування системного промпта (методологія інтерв'ю) — головне IP.
- `POST /api/chat` із викликом Claude, сесіями, лімітами.
- Мінімальний інтерфейс для тесту якості діалогу.
- **Результат:** можна провести справжню діагностичну розмову, AI ставить розумні питання по сфері.

### Фаза 2 — Генерація структурованого звіту
- JSON-schema structured output для аудиту.
- Логіка «достатньо інформації → генеруємо звіт».
- **Результат:** розмова завершується структурованим аудитом (поки сирий/базовий рендер).

### Фаза 3 — Frontend-продукт (вигляд)
- Брендований лендинг + чат-UI + сторінка звіту. Полірований, відмінний від «AI-шаблону» дизайн.
- **Результат:** виглядає як справжній продукт end-to-end на екрані.

### Фаза 4 — Збір і маршрутизація лідів
- Lead gate перед звітом, збереження ліда+сесії в БД, сповіщення Yura (Telegram + email).
- **Результат:** ліди + звіти долітають до тебе.

### Фаза 5 — Захист і контроль витрат
- Turnstile, rate-limit по IP, enforce лімітів, бюджетний cap, моніторинг.
- **Результат:** безпечно слати незнайомцям/у рекламу.

### Фаза 6 — Поліровка, деплой, запуск
- Домен, production-деплой, аналітика, фінальний QA, перший реальний тест із бізнесменом-замовником.
- **Результат:** живе посилання, готове до поширення.

### Беклог (пізніше)
PDF-експорт звіту · мультимова · A/B промпта · дашборд лідів · інтеграція з CRM · ретаргетинг.

## 11. Критерії успіху

- Бізнес-користувач проходить діалог до кінця без плутанини.
- Звіт виглядає переконливо й містить конкретні, релевантні до сфери можливості.
- Кожна можливість має готовий «запит до спеціаліста».
- Лід + звіт надійно долітають до Yura.
- Безпечно для масового поширення (боти/витрати під контролем).
- Вартість одного ліда — копійки.
